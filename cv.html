Models of Interaction (Chapter 3.2)

Interaction involves at least two participants: the user and the system. The interface must effectively translate between them to allow the interaction to be successful. The use of models of interaction can help us understand what is going on in the interaction and identify the likely root of difficulties.

Reference: Dix et al., p. 124-125

Norman's Execution–Evaluation Cycle (p. 125-127)

Norman’s model of interaction is perhaps the most influential in HCI. The user formulates a plan of action, which is then executed at the computer interface. When the plan has been executed, the user observes the computer interface to evaluate the result.

The seven stages are:

  1. Establishing the goal
  2. Forming the intention
  3. Specifying the action sequence
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the system state with respect to the goals and intentions

Gulf of Execution: The difference between the user’s formulation of the actions to reach the goal and the actions allowed by the system.

Gulf of Evaluation: The distance between the physical presentation of the system state and the expectation of the user.

Reference: Dix et al., p. 125-127

Interaction Framework (p. 127-130)

An extension of Norman’s model includes the system explicitly and breaks interaction into four main components: the System, the User, the Input and the Output. Each component has its own language.

There are four main translations involved:

  • Articulation: User's task → Input language
  • Performance: Input language → System operations
  • Presentation: System state → Output
  • Observation: Output → User's interpretation

The framework helps judge the overall usability of an entire interactive system.

Reference: Dix et al., p. 127-130

Interaction Styles (Chapter 3.5)

Interaction can be seen as a dialog between the computer and the user. The choice of interface style can have a profound effect on the nature of this dialog.

Command Line Interface (p. 137)

Provides a means of expressing instructions to the computer directly. Powerful and flexible but requires memorization of commands. Better for expert users than novices.

Menus (p. 137-138)

The set of options available to the user is displayed on the screen. Less demanding of the user, relying on recognition rather than recall. Options need to be meaningfully grouped.

WIMP Interface (p. 141)

Stands for Windows, Icons, Menus and Pointers. The default interface style for most interactive computer systems today (e.g., Microsoft Windows, MacOS).

Other Styles

  • Natural language: Using everyday language (p. 138-139)
  • Form-fills: Resembles paper forms for data entry (p. 139-140)
  • Spreadsheets: Grid of cells containing values or formulas (p. 140-141)
  • Point-and-click: Single-click interactions, common in web browsers (p. 142)
  • Three-dimensional interfaces: Using 3D effects and VR (p. 142-144)

Reference: Dix et al., p. 136-144

Book Reference
