The GOMS Model (Chapter 12.2)

The GOMS model of Card, Moran and Newell is an acronym for Goals, Operators, Methods and Selection. A GOMS description consists of these four elements for analyzing user tasks.

Reference: Dix et al., p. 422-424

GOMS Components

Goals: The user’s goals, describing what the user wants to achieve. In GOMS, goals represent a 'memory point' for the user.

Operators: The lowest level of analysis. Basic actions that the user must perform in order to use the system.

Methods: Different ways in which a goal can be split into subgoals. For example, closing a window via menu vs. function key.

Selection: Rules that predict which methods will be used, depending on the user and system state.

Example GOMS structure:

GOAL: ICONIZE-WINDOW
. [select GOAL: USE-CLOSE-METHOD
. . MOVE-MOUSE-TO-WINDOW-HEADER
. . POP-UP-MENU
. . CLICK-OVER-CLOSE-OPTION
GOAL: USE-L7-METHOD
. . PRESS-L7-KEY]

Selection rules example:

User Sam:
Rule 1: Use the CLOSE-METHOD unless another rule applies.
Rule 2: If the application is 'blocks' use the L7-METHOD.

GOMS Analysis & Performance

The original GOMS model was good for describing how experts perform routine tasks. Coupled with physical device models, it can be used to predict user performance in terms of execution times.

In early experiments, GOMS achieved about 90% correct prediction of user commands and predicted execution times with an error of 33%.

Analysis of GOMS goal structure can yield measures of performance. The stacking depth can estimate short-term memory requirements.

Reference: Dix et al., p. 423-424

Keystroke-Level Model (KLM)

The Keystroke-Level Model (KLM) is a simplified version of GOMS focused on predicting execution time for routine tasks.

KLM breaks down tasks into primitive operators with estimated times:

  • Keystroking (K): ~0.2 sec
  • Pointing (P): ~1.1 sec
  • Homing (H): ~0.4 sec
  • Mental preparation (M): ~1.35 sec
  • Response time (R): system-dependent

Total time = Sum of all operator times

Comparison: GOMS vs. KLM

Aspect GOMS KLM
Full Name Goals, Operators, Methods, Selection Keystroke-Level Model
Complexity More complex, detailed analysis Simplified, focused on time prediction
Focus Task structure, user knowledge, goal hierarchy Execution time, physical/motor operations
Best For Understanding how experts perform routine tasks Predicting time for error-free task completion
Output Goal hierarchies, selection rules, memory requirements Total execution time estimate
Granularity Can be high-level (goals) or low-level (operators) Primitive operators only (K, P, H, M, R)
Relation Original comprehensive model Simplified version of GOMS

Key Difference: GOMS provides a comprehensive analysis of user knowledge and task structure, while KLM focuses specifically on predicting execution time using standardized operator times.
